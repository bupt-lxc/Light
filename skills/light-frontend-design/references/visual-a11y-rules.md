# 视觉可访问性判据（WCAG 阈值 + 栅格 + 触控目标）

> 方法论层 · 跨学科通用、几乎不过时:**对任意视觉设计都成立的硬阈值**集中一处。
> **对比度数学的单一真相源 = `_shared/visual_qa`**(WCAG 2.x 相对亮度/对比度公式 + AA/AAA 阈值);`contrast_lint.py` **复用它不重造**——本文件只列**阈值表供人查**,机检走脚本。
> （**v2 诚实更正**:v1 这里把范式专属 a11y 指向 `db05` 设计卡——一手核 v2 无 db05;范式专属判断(玻璃拟态/新拟物可读性)改为在线找灵感时人判 + ai_tell_lint 抓 slop 痕迹。）
> 权威源：[WCAG 2.2](https://www.w3.org/TR/WCAG22/)、[Apple HIG](https://developer.apple.com/design/human-interface-guidelines/)、[Material Design](https://m3.material.io/)。

## 1. 对比度（WCAG 2.2 AA，硬门槛;机检见 `contrast_lint` → `_shared/visual_qa`）

| 元素 | 最小对比度 | 说明 |
|---|---|---|
| 正文文字（< 18pt 常规 / < 14pt 粗体） | **4.5:1** | 与背景色 |
| 大字（≥ 18pt 常规 / ≥ 14pt 粗体） | **3:1** | |
| UI 组件 / 图形（按钮边界、图标、图表线） | **3:1** | 与相邻色 |
| 装饰性元素 / 失效态 | 无要求 | 不承载信息 |

- AAA 级正文 7:1、大字 4.5:1（投影/答辩/低视力受众建议靠 AAA）。
- 颜色**不作唯一信息载体**：状态除颜色外加图标/文字/形状（红绿色盲约占男性 8%）。
- 校验：**优先用 `scripts/contrast_lint.py`**(抽 design-token/CSS 配色 → 委托 `_shared/visual_qa` 判 AA/AAA,可机检可复现);辅以浏览器 DevTools / WebAIM Contrast Checker 人工复核。落 token 时在项目自有 token 文件里锁定(v2 无 Light 中央 design_tokens)。
- WCAG 2.2 新增 **2.4.11 焦点外观**:焦点指示器 ≥2 CSS px 周长面积 + 焦点态/非焦点态对比 ≥3:1(见 §5)。

## 2. 触控目标尺寸

| 平台 | 最小可点区域 | 来源 |
|---|---|---|
| iOS | **44×44 pt** | Apple HIG |
| Android / Material | **48×48 dp** | Material Design |
| Web（WCAG 2.2 AA 2.5.8） | **24×24 px**（建议 44+） | WCAG 2.2 |

- 相邻目标间距 ≥ 8px，避免误触。移动端主操作放拇指热区（屏幕下 1/3）。

## 3. 间距与栅格（4/8pt 系统）

- 间距、尺寸取 **4 或 8 的倍数**（4/8/12/16/24/32/48/64）——多端一致、易 token 化。
- 正文**行长 45–75 字符**（中文 25–40 字）最易读；行高 1.5–1.7。
- 字体层次少而精：标题/正文/辅助 3–4 级即可，靠字号+字重拉开而非堆字体。

## 4. 动效与运动

- 尊重 `prefers-reduced-motion`：用户开启减弱动效时，关闭视差/自动播放/大位移，保留必要反馈。
- 过渡时长 150–300ms（微交互偏短）；避免闪烁（每秒 > 3 次闪烁有诱发癫痫风险，WCAG 2.3.1）。

## 5. 键盘与焦点

- 所有可交互元素键盘可达（Tab 顺序合理）；焦点态**可见**（不可 `outline:none` 不补替代）。
- 模态打开时焦点移入、关闭时归还；Esc 可关。
