# 文件理解笔记 · {{file_name}}

> 一句话定位：这份文件是什么、为什么要读它、读完要支撑哪个下游动作。

## 0 · 文件元信息

| 字段 | 内容 |
|------|------|
| **文件名/路径** | `{{path}}` |
| **源文件 SHA-256** | `sha256:{{source_sha256}}` |
| **读取契约输入** | `{{reading_contract_path}}` |

| **读取契约报告** | `{{reading_contract_report_path}}` |

| **读取方式** | 宿主原生 Read / pdf_ops / docx_read / xlsx_read / pandoc / markitdown / OCR / ffmpeg+whisper |
| **输入分级** | born-digital / mixed / scanned / sparse-or-unknown / 非 PDF（依据与阈值） |
| **页数/规模** | 页数 / 字数 / 行列 / 时长 |

| **版权/敏感** | 是否受版权全文不外传；是否含密钥/隐私（按 key 名引用，不回显值） |
| **可信度** | 一手/二手、是否权威、是否最新 |
| **读取覆盖** | 已读页/章节/sheet/对象；抽样还是全量 |
| **未读/不可读** | 未覆盖页、加密/损坏、扫描页、缺失附件；没有则写“无” |
| **抽取风险** | OCR/阅读顺序/表格/公式/图片/缓存值风险；对应补读或复核方式 |

> 交下游前，笔记必须绑定当前源文件、读取契约输入和 PASS 报告的原始字节 SHA-256；源文件或契约变化后旧笔记失效。

## 1 · 结构与逻辑（骨架）

- **章节/段落骨架**：列大纲层级（用 `docx_read.py headings` / PDF 目录提取）。
- **论证链 / 叙事骨架**：问题→方法→证据→结论的推进路径，或叙事顺序。
- **章节关系**：哪节支撑哪节、依赖/递进/对比关系。
- **跨文件关系**（批处理时填）：本文件与其他材料的引用/补充/冲突关系。

## 2 · 关键内容（讲了什么）

| 维度 | 内容 |
|------|------|
| **核心问题** | 这份文件要解决/回答什么 |
| **方法/做法** | 用了什么方法、流程、设计 |
| **数据** | 关键数据点、数据集、样本量、统计口径（每条带页/节/表或 sheet+单元格定位） |
| **结果** | 主要发现、指标、结论性数字（每条带页/节/图表定位） |
| **结论/主张** | 作者的核心 claim、证据定位与边界条件 |
| **未决/存疑** | 缺失信息、未验证主张、需进一步核实之处 |

## 3 · 格式与要求（模板/约束）

- **模板规范**：页边距/纸张/字号/行距/编号（来自 `docx_read.py layout` / `read_runs`）。
- **引用风格**：APA / GB-T 7714 / IEEE 等；参考文献格式。
- **字数/篇幅约束**：投稿/任务的硬上限。
- **隐含要求**：从文件推断但未明说的约束（如盲审要求、术语统一、占位符残留）。

## 4 · 视觉风格（供 frontend-design / figure）

- **配色**：主色/辅色、是否成体系（截图取色）。
- **版式**：栏数、留白、对齐、标题层级（PPT 36-44pt / 正文 14-16pt 量级）。
- **图表风格**：图型、标注规范、误差棒/单位/panel 标号习惯。
- **可沉淀去向**：前端/界面风格→frontend-design、图表风格→figure；**v2 无本地设计库，风格直接喂对应下游技能**（当场在线找灵感而非吃陈旧本地库）。

## 5 · 可复用内容（能直接拿来用的）

| 类型 | 位置 | 复用去向 |
|------|------|----------|
| 段落/表述 | 第X节/页 | 论文/报告改写 |
| 数据/表格 | 表X/sheet X | 重绘图表 / 数据分析 |
| 图/示意 | 图X | 复用或重绘 |
| 结构/模板 | 全文骨架 | 套用为新文档模板 |

### 5.1 · 定位与覆盖账

| 对象 | 定位 | 读取方式/置信 | 状态与缺口 |
|------|------|-------------|-----------|
| claim / 数字 / 表 / 图 | PDF 页+节/图表号；DOCX 标题+段落；XLSX sheet+单元格 | 原生文本 / OCR / 视觉 / 表抽取 | 已核 / 待视觉核 / 未覆盖 |

## 6 · 文件 → 下游技能动作映射（转化为可执行任务）

> 明确"这个文件→接下来能做什么"，每条给出触发条件与目标技能。

|----------|----------|----------|
