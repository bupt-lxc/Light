# Human review of `issue-matrix.json`

> The JSON issue matrix is canonical. Copy reviewer wording exactly. Do not
> change `PLANNED` to `DONE` until the owning skill produces a real locator and,
> for experiments/analysis, run provenance.

| issue_id | reviewer/round | verbatim raw span | type | root cause | rejection-driving evidence | claim IDs | evidence IDs | strategy | owner | status | change locator | response |
|---|---|---|---|---|---|---|---|---|---|---|---|---|
|  |  |  | request / claim / question / misunderstanding / editorial |  | NONE or kind+value+locator |  |  | acknowledge_and_fix / rebut_with_evidence / clarify / downgrade_claim / request_editor_ruling |  | PLANNED / IN_PROGRESS / DONE / DECLINED / NOT_APPLICABLE |  |  |

Check:

- raw span occurs verbatim in the registry source text;
- major does not imply critical without rejection-driving evidence;
- reviewer error is answered with evidence or editor escalation, not ignored;
- planned work is not described as completed;
- DONE points to the actual revised artifact;
- new citations are confirmed by citation;
- new experiment/analysis DONE has run provenance.
